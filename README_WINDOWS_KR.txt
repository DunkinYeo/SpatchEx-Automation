============================================================
  SpatchEx UAT 도구  —  Windows 설치 및 사용 가이드
============================================================

이 도구는 S-Patch EX 앱의 장기 테스트(증상 자동 주입)를
자동화하는 도구입니다. 개발 지식 없이도 사용할 수 있습니다.


------------------------------------------------------------
  시작 전 반드시 확인 — ZIP 파일 차단 해제
------------------------------------------------------------

이메일이나 공유 폴더에서 받은 ZIP 파일은
Windows 보안 정책에 의해 차단될 수 있습니다.

해제 방법:
  1) ZIP 파일을 마우스 오른쪽 버튼으로 클릭
  2) [속성] 선택
  3) 창 하단에 "이 파일은 다른 컴퓨터에서 가져온 것입니다..."
     라는 문구와 [차단 해제] 체크박스가 있으면 체크 후 [확인]
  4) 이후 ZIP을 압축 해제하여 사용

※ 이 단계를 건너뛰면 install.bat 실행 시
  "Windows가 PC를 보호했습니다" 오류가 발생할 수 있습니다.


------------------------------------------------------------
  1. 최초 1회 설치
------------------------------------------------------------

  1) 안드로이드 스마트폰을 USB 케이블로 PC에 연결합니다.

  2) 스마트폰에서 USB 디버깅을 활성화합니다.
       설정 → 휴대전화 정보 → 소프트웨어 정보
       → [빌드 번호]를 7번 빠르게 탭
       설정 → 개발자 옵션 → USB 디버깅 ON

  3) 스마트폰에 "USB 디버깅을 허용하시겠습니까?" 팝업이 뜨면
     [허용]을 탭합니다.

  4) install.bat 을 더블클릭합니다.
     Python, Node.js, ADB, Appium, Python 패키지가 자동 설치됩니다.
     (최초 실행 시 5~10분 소요됩니다.)

  ※ CMD 창에서 진행이 멈춘 것처럼 보일 때는
    Enter 키를 한두 번 눌러보세요.
    실제로는 설치가 진행 중인 경우가 많습니다.


------------------------------------------------------------
  2. 테스트 시작
------------------------------------------------------------

  1) S-Patch EX 앱을 켜고 ECG 측정이 진행 중인지 확인합니다.

  2) run.bat 을 더블클릭합니다.
     잠시 후 브라우저가 자동으로 열립니다.

  3) 브라우저에서:
     - 기기 목록에서 연결된 디바이스를 선택합니다.
     - 실행 이름을 입력합니다 (선택 사항).
     - [테스트 시작] 버튼을 클릭합니다.

  4) 테스트가 진행되는 동안 run.bat 창을 닫지 마세요.


------------------------------------------------------------
  3. 테스트 중지
------------------------------------------------------------

  STOP.bat 을 더블클릭합니다.
    — 또는 —
  브라우저에서 [테스트 중지] 버튼을 클릭하세요.


------------------------------------------------------------
  4. 문제 해결
------------------------------------------------------------

  기기가 인식되지 않는 경우:
    - USB 케이블을 뽑았다가 다시 꽂아 보세요.
    - 스마트폰의 USB 디버깅 설정을 다시 확인하세요.
    - 스마트폰 화면에 "허용" 팝업이 있으면 탭하세요.

  install.bat 실행 중 오류가 발생하는 경우:
    - install.bat 을 다시 실행하면 대부분 해결됩니다.

  브라우저가 열리지 않는 경우:
    - 수동으로 접속하세요:  http://127.0.0.1:5001

  테스트가 계속 실패하는 경우:
    - 브라우저의 [Failures] 탭에서 스크린샷과 오류 내용을 확인하세요.


------------------------------------------------------------
  5. 팀 대시보드 (선택 사항)
------------------------------------------------------------

  다른 PC에서 테스트 진행 상황을 모니터링하려면
  README_TEAM_DASHBOARD_KR.txt 를 참고하세요.
