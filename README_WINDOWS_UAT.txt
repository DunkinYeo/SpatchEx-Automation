=============================================
  SpatchEx Automation — Windows UAT 실행 가이드
=============================================

이 문서는 QA/CS UAT 참가자를 위한 안내문입니다.
개발 지식 없이도 따라할 수 있습니다.


---------------------------------------------
  1. 사전 준비
---------------------------------------------

아래 항목을 미리 확인해주세요.

  - Windows 10 이상
  - 인터넷 연결 (최초 설치 시 필요)
  - Android 스마트폰
  - USB 케이블 (데이터 전송 지원)
  - USB 디버깅 활성화 (아래 방법 참고)


  [ USB 디버깅 활성화 방법 ]

  1) 스마트폰에서 [설정] 앱을 엽니다.
  2) [휴대전화 정보] 또는 [디바이스 정보]를 탭합니다.
  3) [소프트웨어 정보]를 탭합니다.
  4) [빌드 번호]를 7번 빠르게 탭합니다.
     "개발자 옵션이 활성화되었습니다" 메시지가 나타납니다.
  5) [설정]으로 돌아가 [개발자 옵션]을 탭합니다.
  6) [USB 디버깅]을 켭니다.
  7) PC에 USB로 연결하면 폰 화면에 허용 팝업이 나타납니다.
     [허용]을 탭합니다.


---------------------------------------------
  2. ZIP 파일 압축 해제
---------------------------------------------

  1) 전달받은 ZIP 파일을 다운로드합니다.

  2) ZIP 파일을 마우스 오른쪽 버튼으로 클릭하고
     [속성]을 선택합니다.

     속성 창 하단에 "보안" 항목이 보이면:
       [차단 해제] 체크박스에 체크한 뒤 [확인]을 클릭합니다.

     ※ 이 단계를 건너뛰면 install.bat 실행 시
        "Windows가 PC를 보호했습니다" 오류가 발생할 수 있습니다.

  3) ZIP 파일을 마우스 오른쪽 버튼으로 클릭하고
     [모두 압축 해제]를 선택합니다.

  4) 압축이 풀린 [SpatchEx-Automation] 폴더를 엽니다.


---------------------------------------------
  3. 설치 실행 (최초 1회만)
---------------------------------------------

  1) [SpatchEx-Automation] 폴더 안의
     [install.bat] 파일을 더블 클릭합니다.

  2) 검은 창(명령 프롬프트)이 열리며 설치가 진행됩니다.
     아래 항목들이 자동으로 처리됩니다.

       - Python 환경 확인
       - Node.js 환경 확인
       - Appium 서버 설치
       - UiAutomator2 드라이버 설치
       - Python 패키지 설치

  3) 설치는 인터넷 속도에 따라 5~10분 걸릴 수 있습니다.
     창을 닫지 말고 완료될 때까지 기다려주세요.

  4) 설치가 완료되면 아래 메시지가 표시됩니다.

       SpatchEx Automation Installed

     [아무 키나 누르면] 창이 닫힙니다.

  ※ 오류 메시지가 나타나면 담당자에게 화면을 캡처해서 공유해주세요.


---------------------------------------------
  4. Android 기기 연결
---------------------------------------------

  1) USB 케이블로 스마트폰을 PC에 연결합니다.
  2) 스마트폰 화면에 [USB 디버깅을 허용하시겠습니까?] 팝업이
     나타나면 [허용]을 탭합니다.
  3) 테스트 중에는 화면 잠금이 풀린 상태를 유지해주세요.


---------------------------------------------
  5. 테스트 실행
---------------------------------------------

  1) [SpatchEx-Automation] 폴더 안의
     [run.bat] 파일을 더블 클릭합니다.

  2) 다음이 자동으로 시작됩니다.

       - Appium 서버 (기기 제어용)
       - 웹 테스트 인터페이스
       - 브라우저 자동 실행

  3) 브라우저가 자동으로 열리며 테스트 화면이 표시됩니다.
     브라우저가 열리지 않으면 아래 6번을 참고하세요.

  ※ run.bat 창은 테스트가 진행되는 동안 열어두어야 합니다.
     창을 닫으면 테스트가 중단됩니다.


---------------------------------------------
  6. 웹 테스트 화면 접속
---------------------------------------------

  브라우저에서 아래 주소로 직접 접속할 수 있습니다.

    http://127.0.0.1:5001


---------------------------------------------
  7. 테스트 종료
---------------------------------------------

  테스트가 끝나면 [STOP.bat] 파일을 더블 클릭합니다.
  모든 서비스가 종료됩니다.


---------------------------------------------
  8. 문제 해결
---------------------------------------------

  [ 기기가 인식되지 않는 경우 ]

  1) 명령 프롬프트(CMD)를 엽니다.
     시작 메뉴 → [cmd] 검색 → [명령 프롬프트] 실행
  2) 아래 명령어를 입력하고 Enter를 누릅니다.

       adb devices

  3) 결과에 기기 번호와 [device] 가 표시되면 정상입니다.
     [unauthorized] 가 표시되면 폰에서 USB 디버깅 허용 팝업을
     다시 확인해주세요.

  [ USB 디버깅이 보이지 않는 경우 ]

  - 제조사에 따라 [개발자 옵션] 메뉴 위치가 다를 수 있습니다.
  - [빌드 번호] 7회 탭이 적용되지 않은 경우 다시 시도해주세요.
  - 삼성 기기: 설정 → 디바이스 정보 → 소프트웨어 정보 → 빌드 번호
  - 픽셀 기기: 설정 → 휴대전화 정보 → 빌드 번호

  [ 그 외 문제 ]

  - 오류 화면을 캡처하여 담당자에게 공유해주세요.


---------------------------------------------
  9. QA 워크플로우 요약
---------------------------------------------

  [ 완전 초기화 후 재설치 (UAT 시뮬레이션) ]

    clean_env.bat   -- .venv, logs, artifacts 전체 삭제 후 폴더 재생성
    install.bat     -- 환경 재설치
    run.bat         -- 테스트 실행

  [ 빠른 재실행 (로컬 반복 테스트) ]

    quick_clean.bat -- logs, artifacts 만 초기화 (.venv 유지)
    run.bat         -- 테스트 실행

  [ 환경 진단만 필요한 경우 ]

    selftest.bat    -- Python, ADB, Appium, 패키지 상태 확인
