============================================================
  SpatchEx UAT Tool  --  WiFi 기기 연결 가이드
============================================================

USB 케이블 없이 Wi-Fi로 SpatchEx를 실행하는 방법을 설명합니다.

Wi-Fi로 연결된 후에는 USB 연결과 동일하게 모든 기능이
정상 동작합니다.


------------------------------------------------------------
  이 방법을 사용하는 경우
------------------------------------------------------------

  다음과 같은 상황에서 WiFi 모드를 사용하세요:
  - USB 케이블이 없거나 불편한 경우
  - 피검자가 기기를 착용한 채로 움직여야 하는 경우
  - 케이블 없이 테스트를 진행하고 싶은 경우

  USB 모드(run.bat)는 기존과 동일하게 유지됩니다.


------------------------------------------------------------
  준비사항
------------------------------------------------------------

  - Android 기기와 PC가 동일한 Wi-Fi 네트워크에 연결되어 있어야 합니다
  - 기기에서 USB 디버깅이 활성화되어 있어야 합니다
  - Android 11 이상: 무선 디버깅을 바로 사용할 수 있습니다
  - Android 10 이하: 초기 설정 시 USB 연결이 필요합니다


------------------------------------------------------------
  최초 설정  (Android 10 이하)
------------------------------------------------------------

  1단계 -- USB로 연결
    USB 케이블로 Android 기기를 PC에 연결합니다.

  2단계 -- ADB TCP/IP 모드 활성화
    명령 프롬프트에서 다음을 실행합니다:

      adb tcpip 5555

    "restarting in TCP mode port: 5555" 메시지가 표시됩니다.

  3단계 -- 기기의 IP 주소 확인
    기기에서:
      설정 -> 휴대전화 정보 -> 상태 정보 -> IP 주소
      (또는 설정 -> Wi-Fi -> 연결된 네트워크 탭)

  4단계 -- USB 케이블 분리
    이제 USB 케이블을 뽑아도 됩니다.

  5단계 -- Wi-Fi로 연결
    다음을 실행합니다:

      adb connect 192.168.0.50:5555

    (실제 기기 IP 주소로 변경)

  이 설정은 기기 재시작 후마다 2단계부터 다시 해야 합니다.


------------------------------------------------------------
  최초 설정  (Android 11 이상, 무선 디버깅)
------------------------------------------------------------

  기기에서:
    설정 -> 개발자 옵션 -> 무선 디버깅 -> 켜기
    "페어링 코드로 기기 페어링" 탭
    화면에 표시된 IP 주소와 포트 메모

  PC에서:
    adb pair <ip>:<페어링 포트>
    (기기에 표시된 6자리 코드 입력)

    연결:
    adb connect <ip>:5555


------------------------------------------------------------
  WiFi로 테스트 실행하기
------------------------------------------------------------

  run_wifi.bat 더블클릭

  1. 기기 IP:Port를 입력하라는 메시지가 나타납니다:

       Enter Android device IP:Port
       Example: 192.168.0.50:5555

  2. 기기의 IP 주소와 포트(기본값: 5555)를 입력합니다.

  3. 연결 확인 후 자동으로 테스트 환경이 실행됩니다.
     (run.bat와 동일한 방식)

  4. 브라우저가 자동으로 열립니다.
     이후 USB 연결과 동일하게 진행합니다.


------------------------------------------------------------
  웹 UI에서 WiFi 연결  (선택사항)
------------------------------------------------------------

  웹 UI에서도 직접 WiFi 연결을 설정할 수 있습니다:

  1. 브라우저 열기 (http://127.0.0.1:5001)
  2. "Device Connection"에서 "WiFi ADB" 선택
  3. Device Address 입력: 192.168.0.50:5555
  4. Start Test 클릭

  테스트 시작 전 자동으로 adb connect가 실행됩니다.


------------------------------------------------------------
  문제 해결
------------------------------------------------------------

  "기기를 찾을 수 없음" 또는 "인증되지 않음"?
    - 기기와 PC가 같은 Wi-Fi 네트워크에 있는지 확인
    - USB 연결 상태에서 adb tcpip 5555 재실행
    - 기기에서 IP 주소 다시 확인
    - 기기에 팝업이 나타나면 "허용" 탭

  테스트 중 연결이 끊기는 경우?
    - 화면 자동 꺼짐 비활성화 (개발자 옵션)
    - Wi-Fi 절전 모드 비활성화
    - 비행기 모드 전환 방지

  USB로 돌아가려면?
    - run.bat를 그대로 사용하면 됩니다
    - USB 모드는 WiFi 모드와 완전히 독립적입니다
