=============================================
  SpatchEx Automation — macOS UAT 실행 가이드
=============================================

이 문서는 QA/CS UAT 참가자를 위한 안내문입니다.
개발 지식 없이도 따라할 수 있습니다.


---------------------------------------------
  1. 사전 준비
---------------------------------------------

아래 항목을 미리 확인해주세요.

  - macOS 12 (Monterey) 이상
  - 인터넷 연결 (최초 설치 시 필요)
  - Android 스마트폰
  - USB 케이블 (데이터 전송 지원)
  - USB 디버깅 활성화 (아래 방법 참고)


  [ USB 디버깅 활성화 방법 ]

  1) 스마트폰에서 [설정] 앱을 엽니다.
  2) [휴대전화 정보] 또는 [디바이스 정보]를 탭합니다.
  3) [소프트웨어 정보]를 탭합니다.
  4) [빌드 번호]를 7번 빠르게 탭합니다.
     "개발자 옵션이 활성화되었습니다" 메시지가 나타납니다.
  5) [설정]으로 돌아가 [개발자 옵션]을 탭합니다.
  6) [USB 디버깅]을 켭니다.
  7) Mac에 USB로 연결하면 폰 화면에 허용 팝업이 나타납니다.
     [허용]을 탭합니다.


---------------------------------------------
  2. ZIP 파일 압축 해제
---------------------------------------------

  1) 전달받은 ZIP 파일을 다운로드합니다.
  2) ZIP 파일을 더블 클릭하면 자동으로 압축이 해제됩니다.
  3) 압축이 풀린 [SpatchEx-Automation] 폴더를 엽니다.


---------------------------------------------
  3. 설치 실행 (최초 1회만)
---------------------------------------------

  1) [런치패드] 또는 [응용 프로그램 → 유틸리티]에서
     [Terminal] (터미널)을 실행합니다.

  2) 아래 명령어를 순서대로 입력합니다.
     (각 줄을 입력한 후 Enter를 누르세요)

       cd ~/Downloads/SpatchEx-Automation

       chmod +x install.command

       ./install.command

  3) 터미널 창에서 설치가 진행됩니다.
     아래 항목들이 자동으로 처리됩니다.

       - Python 환경 확인
       - Node.js 환경 확인
       - Appium 서버 설치
       - UiAutomator2 드라이버 설치
       - Python 패키지 설치

  4) 설치는 인터넷 속도에 따라 5~10분 걸릴 수 있습니다.
     창을 닫지 말고 완료될 때까지 기다려주세요.

  5) 설치가 완료되면 아래 메시지가 표시됩니다.

       SpatchEx Automation Installed

     Enter를 눌러 창을 닫습니다.

  ※ 오류 메시지가 나타나면 담당자에게 화면을 캡처해서 공유해주세요.

  [ Finder에서 바로 실행하는 방법 ]

  터미널 명령어가 어렵다면, Finder에서 install.command 파일을
  마우스 오른쪽 버튼으로 클릭하고 [열기]를 선택하세요.
  처음 실행 시 보안 경고가 나타나면 [열기]를 다시 클릭합니다.


---------------------------------------------
  4. Android 기기 연결
---------------------------------------------

  1) USB 케이블로 스마트폰을 Mac에 연결합니다.
  2) 스마트폰 화면에 [USB 디버깅을 허용하시겠습니까?] 팝업이
     나타나면 [허용]을 탭합니다.
  3) 테스트 중에는 화면 잠금이 풀린 상태를 유지해주세요.


---------------------------------------------
  5. 테스트 실행
---------------------------------------------

  방법 1 — Finder에서 실행 (권장)

    [SpatchEx-Automation] 폴더 안의 [run.command] 파일을
    더블 클릭합니다.

  방법 2 — 터미널에서 실행

    터미널에서 아래 명령어를 입력합니다.

      ./run.command

  실행되면 다음이 자동으로 시작됩니다.

    - Appium 서버 (기기 제어용)
    - 웹 테스트 인터페이스
    - 브라우저 자동 실행

  브라우저가 자동으로 열리며 테스트 화면이 표시됩니다.
  브라우저가 열리지 않으면 아래 6번을 참고하세요.

  ※ run.command 창은 테스트가 진행되는 동안 열어두어야 합니다.
     창을 닫으면 테스트가 중단됩니다.


---------------------------------------------
  6. 웹 테스트 화면 접속
---------------------------------------------

  브라우저에서 아래 주소로 직접 접속할 수 있습니다.

    http://127.0.0.1:5001


---------------------------------------------
  7. 테스트 종료
---------------------------------------------

  방법 1 — STOP.command 실행

    [SpatchEx-Automation] 폴더 안의 [STOP.command] 파일을
    더블 클릭합니다.

  방법 2 — 터미널에서 종료

    run.command 가 실행 중인 터미널 창에서
    Ctrl + C 를 누릅니다.
