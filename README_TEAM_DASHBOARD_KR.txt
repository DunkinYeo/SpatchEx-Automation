============================================================
  SpatchEx UAT 도구  —  팀 대시보드 사용 가이드
============================================================


------------------------------------------------------------
  이 기능이 필요한지 먼저 확인하세요
------------------------------------------------------------

  필요 없음 (아래 해당하면):
    - 혼자 테스트를 진행하는 경우
    - 사내 네트워크 밖에서 테스트하는 경우
    → Hub URL 칸을 비워두면 됩니다.
      테스트는 정상적으로 실행됩니다.

  필요함 (아래 해당하면):
    - QA 리드 또는 담당자가 여러 테스터의
      진행 상황을 한 화면에서 모니터링하고 싶은 경우


------------------------------------------------------------
  동작 방식
------------------------------------------------------------

  [대시보드 PC]  — 모든 테스터가 공유하는 모니터링 화면

    1) 대시보드 PC에서 run.bat (Windows) 또는
       run.command (Mac) 를 실행합니다.

    2) 대시보드 PC의 로컬 IP 주소를 확인합니다.
         Windows: 명령 프롬프트 → ipconfig 입력
         Mac:     터미널 → ifconfig | grep "inet " 입력

    3) run.bat 창 또는 run.command 창을 열어 둡니다.

    4) 브라우저에서 팀 뷰를 엽니다:
         http://<대시보드PC의IP주소>:5001/team
         예시:  http://192.168.0.4:5001/team


  [테스터 PC]  — 실제 테스트를 실행하는 각 테스터

    1) 자신의 PC에서 run.bat 또는 run.command 를 실행합니다.

    2) 브라우저에 열리는 화면에서:
         Hub URL:    http://192.168.0.4:5001
                     (대시보드 PC의 IP 주소 입력)
         테스터 이름: 본인의 이름 또는 ID 입력

    3) [테스트 시작] 버튼을 클릭합니다.

  이후 대시보드 PC의 /team 페이지에서
  각 테스터의 진행 상황이 실시간으로 보입니다.


------------------------------------------------------------
  주의 사항
------------------------------------------------------------

  · 대시보드 PC와 테스터 PC는 같은 Wi-Fi 또는 사내 LAN에
    연결되어 있어야 합니다.
    인터넷(외부 네트워크)으로는 동작하지 않습니다.

  · 대시보드 PC의 run.bat / run.command 가 먼저 실행 중이어야
    테스터의 데이터가 수신됩니다.

  · 테스터마다 다른 이름을 사용해야 대시보드에서
    개별 카드로 구분됩니다.

  · Hub URL 을 입력하지 않으면 테스트는 정상 실행됩니다.
    원격 리포팅만 생략될 뿐입니다.


------------------------------------------------------------
  대시보드에서 볼 수 있는 화면
------------------------------------------------------------

  /team       — 테스터별 카드 (상태, 기기, 마지막 이벤트)
  /failures   — 실패 스크린샷 + 오류 로그
  /timeline   — 전체 이벤트 타임라인
